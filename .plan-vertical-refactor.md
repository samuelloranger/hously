# Plan: Vertical Codebase Reorganization

**Goal:** Eliminate the horizontal `apps/web/src/hooks/` directory by colocating domain hooks
into their feature verticals. Move web-only files out of `apps/shared/` into `apps/web/`.

**Principle:** Code that changes together lives together. `lib/` is for true infrastructure
(auth, realtime, app-wide utilities). Feature pages own their data hooks. Shared owns only
code the API or cross-app contracts require.

---

## Phase 2 — Move shared endpoints into web

**Why:** Endpoint URL constants are web-only (the iOS app is Swift and cannot import TypeScript;
the API does not route to itself via these strings). One exception: `QBITTORRENT_TORRENTS_PAGE_SIZE`
from `endpoints/qbittorrent.ts` is imported by `apps/api/`.

### Files to move: `apps/shared/src/endpoints/` → `apps/web/src/lib/endpoints/`

Move all 18 endpoint files. Keep folder flat, mirroring current structure.

**Exception — preserve API constant:**
Before moving `endpoints/qbittorrent.ts`, extract `QBITTORRENT_TORRENTS_PAGE_SIZE` into
`apps/shared/src/constants/qbittorrent.ts` and update the API import accordingly. Then the
endpoint file can safely move to web.

### Import path changes in `apps/web/src/`

- Old: `import { CHORES_ENDPOINTS } from '@hously/shared'`
- New: `import { CHORES_ENDPOINTS } from '@/lib/endpoints/chores'`

Or create `apps/web/src/lib/endpoints/index.ts` that re-exports all of them for a single
import target.

### Update `apps/shared/src/index.ts`

Remove all endpoint re-exports.

## Risk notes

- **Test imports**: Tests in `__tests__/` directories import hooks directly — update alongside the hook move in each phase.
- **Cross-boundary imports**: After Phase 3, `pages/_component/HomePanel.tsx` will import from `pages/chores/useChores.ts` and `pages/habits/useHabits.ts`. This is acceptable — the home page is an aggregator.
- **Barrel file hygiene**: When creating `features/*/hooks/index.ts` files, only re-export what is public. Don't accidentally re-export internal helpers.
- **Path aliases**: `@/` resolves to `apps/web/src/` — all new `lib/` and `pages/` paths are reachable via this alias.
