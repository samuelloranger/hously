# Plan: Hacker News Dashboard Widget Integration

## Overview

Add a Hacker News feed widget to the Hously dashboard. Since the HN API is fully public (no auth, no rate limit), this integration doesn't need a plugin with credentials ‚Äî just an enabled/disabled toggle via the existing plugin system, plus a configurable story count and feed type (top/best/new/ask/show).

The API backend will fetch stories from `hacker-news.firebaseio.com`, cache them in Redis (5-minute TTL), and serve them to the frontend as a dashboard widget.

## Files to Create

### 1. API ‚Äî Utility function
**`apps/api/src/utils/dashboard/hackernews.ts`**
- `fetchHackerNewsStories()` function
- Check plugin enabled via `prisma.plugin.findFirst({ where: { type: 'hackernews' } })`
- Read config: `feed_type` (top/best/new/ask/show), `story_count` (default 10, max 30)
- Check Redis cache (`dashboard:hackernews:{feed_type}`, TTL 5 min)
- On miss: fetch story IDs from `https://hacker-news.firebaseio.com/v0/{feed_type}stories.json`
- Fetch individual items in parallel (batch of `story_count` items via `Promise.all`)
- Map to `HackerNewsStory` shape: `{ id, title, url, score, by, time, comment_count, type }`
- Cache result and return `DashboardHackerNewsResponse`

### 2. API ‚Äî Dashboard route
**`apps/api/src/routes/dashboard/hackernewsRoutes.ts`**
- `GET /hackernews` ‚Äî calls `fetchHackerNewsStories()`, returns response
